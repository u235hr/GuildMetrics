# 响应式布局规范

## 基础布局要求

1. 布局技术选型
   - 必须使用 CSS Grid 或 Flexbox 
   - 确保组件间不发生重叠和遮挡

2. 尺寸单位规范
   - 使用相对单位: %, vh, vw, fr, em, rem
   - 严禁使用固定像素值(px)
   - 字体和组件大小通过 clamp() 或 vw/vh 实现自适应

3. 容器规范
   - 容器需设置合理的高度限制
   - 内容溢出时启用垂直滚动 (overflow-y: auto)

4. 兼容性要求
   - 支持多种分辨率: 1920×1080, 1440×900, 4K, 笔记本小屏
   - 窗口缩放时元素应自动换行、缩小或滚动，避免溢出和遮挡

## 主播数据展示布局

1. 顶部区域
   - 水平排列展示前三名主播卡片
   - 一二三名卡片尺寸需有层次区分
   - 小屏下卡片自动缩放或换行

2. 列表区域
   - 固定高度容器承载其他主播列表
   - 启用容器内部滚动
   - 列表区域使用 flex:1 填充剩余空间
   - overflow-y: auto 实现局部滚动

3. 技术实现细节
   - 采用 CSS Grid 布局系统
   - 禁止固定宽高
   - 字体和卡片大小需随屏幕动态调整
   - 超出容器时自动显示滚动条

## 视觉验证规范

- 样式相关修改必须通过截图进行视觉验证
- 禁止仅依赖代码检查和测试验证
- 重点验证自适应效果和滚动行为
