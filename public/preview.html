Centering Layout Demonstration

This shows why a container needs an explicit width for `transform: translateX(-50%)` to work correctly.

1. Your Approach (Incorrect Centering)

The container `div` has `height` but no `width`.

My container has no width. Centering fails.

2. Correct Approach (Correct Centering)

The container `div` has both `height` and `width`.

My container has width. Centering works.

Explanation

The RED box demonstrates your logic. Its light-red container (`wrapper-a`) has a set height but no width. The `transform: translateX(-50%)` needs to know the container's width to center it. Since the width is not defined, the browser cannot calculate "50% of its own width" correctly, so the centering fails. The container simply gets stretched by the card inside it, but it's too late for the `transform` calculation.

The GREEN box shows the fix. Its light-green container (`wrapper-b`) has an explicit `width` (calculated from the height and aspect ratio). Now, the browser knows the exact width, so `transform: translateX(-50%)` can pull it back by the correct amount, achieving a perfect center.
